# تحليل لوحة الإدارة - Admin Dashboard & Management

## Qor3a - System Administration Analysis

---

## 1. من هو الـ Admin؟

**المدير** (Admin) هو مالك/مشغل المنصة. صلاحياته كاملة:

| الصلاحية | الوصف |
|----------|-------|
| 👀 رؤية كل شيء | كل الطلبات، كل المستخدمين، كل المدفوعات |
| 👥 إدارة المستخدمين | إضافة/حذف/تعديل الموظفين والعملاء |
| 💰 التقارير المالية | الإيرادات، المدفوعات، الأرباح |
| 📊 الإحصائيات | عدد الطلبات، نسب الإنجاز، أداء الموظفين |
| 📝 سجل التدقيق | كل حدث في النظام (Audit Log) |
| ⚙️ إعدادات النظام | أسعار الخدمات، حسابات الدفع، API keys |
| 🚨 إدارة الطوارئ | إيقاف/تعليق النظام، حظر مستخدمين |

**المدير لا يقوم بـ:**
- ❌ مراجعة الطلبات (هذا عمل الموظف)
- ❌ تأكيد الدفع (هذا عمل الموظف)
- ❌ إدخال Confirmation# (هذا عمل الموظف)

## 2. هيكل لوحة الإدارة

```
/dashboard/admin
│
├── /                          ← 📊 الرئيسية (الإحصائيات)
│
├── /orders                    ← 📋 كل الطلبات (إشراف)
│   ├── /                      ← قائمة + ترشيح شامل
│   └── /:id                   ← عرض طلب (قراءة فقط - لا تعديل)
│
├── /users                     ← 👥 إدارة المستخدمين
│   ├── /                      ← قائمة + بحث + فلتر
│   ├── /employees             ← الموظفون فقط
│   ├── /clients               ← العملاء فقط
│   ├── /create                ← إضافة موظف جديد
│   └── /:id                   ← عرض/تعديل مستخدم
│
├── /payments                  ← 💰 المدفوعات (سجل كامل)
│   └── /                      ← سجل كل المدفوعات + تصدير
│
├── /reports                   ← 📈 التقارير
│   ├── /revenue               ← تقرير الإيرادات
│   ├── /orders                ← تقرير الطلبات
│   └── /employees             ← تقرير أداء الموظفين
│
├── /audit-log                 ← 📝 سجل التدقيق
│   └── /                      ← كل حدث في النظام
│
├── /settings                  ← ⚙️ إعدادات النظام
│   ├── /general               ← الإعدادات العامة
│   ├── /payment-accounts      ← حسابات الدفع (أرقام حسابات قرعة)
│   ├── /pricing               ← أسعار الخدمات
│   └── /api-keys              ← API keys (2Captcha, SMS, ...)
│
└── /notifications             ← 🔔 إرسال إشعار جماعي
    └── /                      ← إشعار لجميع العملاء/المختارين
```

## 4. مهام الإدارة التشغيلية (Operational Admin Tasks)

### 4.1 إدارة الموظفين (أثناء الموسم)

| المهمة | التكرار | الإجراء |
|--------|---------|---------|
| إضافة موظف جديد | عند الحاجة | الاسم + البريد + صلاحية + كلمة سر |
| إيقاف موظف | عند المخالفة | تعطيل الحساب ← لا يمكنه الدخول |
| متابعة أداء الموظفين | يومياً | عدد الطلبات التي راجعها كل موظف |
| مراجعة الطلبات المعلقة | يومياً | طلبات لم تراجع > 24 ساعة |

### 4.2 إدارة النظام (قبل الموسم)

| المهمة | التوقيت | الإجراء |
|--------|---------|---------|
| تفعيل أسعار الخدمات | قبل الموسم | تحديد سعر التسجيل (1,000 YR) |
| إدخال حسابات الدفع | قبل الموسم | أرقام حسابات كريمي/جيب/ون كاش/موبايل موني |
| اختبار API الإشعارات | قبل الموسم | إرسال إيميل/واتساب تجريبي |
| تفعيل Headless Service | قبل مايو | تشغيل Worker Pool لفحص النتائج |
| تجهيز Proxy Pool | قبل مايو | شراء IPs متعددة للفحص الآلي |

### 4.3 متابعة فحص النتائج (في مايو)

```
يوم إعلان النتائج:
│
├── 08:00  ← تجهيز الـ Workers (10 خوادم)
├── 08:30  ← بدء تشغيل Queue
├── 09:00  ← الإعلان الرسمي ← بدء الفحص
├── 09:00-12:00  ← فحص آلي (100-200 طلب/ساعة)
├── 12:00  ← مراجعة يدوية للفائزين
├── 12:30  ← التواصل مع الفائزين
└── خلال 24 ساعة ← فحص كل الطلبات
```

### 4.4 إدارة الأزمات (Crisis Management)

| المشكلة | الإجراء |
|---------|---------|
| ⚠️ موظف مريض في موسم الذروة | تفعيل موظف احتياطي أو إعادة توزيع الطلبات |
| 🚨 انقطاع الإنترنت | العمل على اتصال بديل (Starlink/4G) |
| 🔴 الموقع الرسمي معطل | تأجيل الإدخال، إشعار العملاء بالتأخير |
| ❌ فشل 2Captcha | تفعيل الطابور اليدوي لحل CAPTCHA |
| 💥 ضغط هائل على الخادم | تفعيل Auto-scaling + Queue |
