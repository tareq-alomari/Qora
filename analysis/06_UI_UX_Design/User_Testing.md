# اختبار المستخدم - User Testing Plan

## Qor3a — كيف نختبر الموقع مع يمنيين حقيقيين؟

---

## 1. لماذا اختبار المستخدم مهم؟

```
في اليمن:
├── 📱 80% يستخدمون جوالات رخيصة
├── 🌐 إنترنت ضعيف
├── 👴 بعض العملاء ليسوا تقنيين
└── 🤝 الثقة أصعب من أي مكان آخر

لذلك:
├── ❌ لا نثق بتجربتنا فقط (نحن مطورون)
├── ❌ لا نطلق مباشرة بدون اختبار
└── ✅ نختبر مع 5-10 مستخدمين حقيقيين قبل الإطلاق
```

---

## 2. من نختبر معه؟

### 2.1 المستخدمون المثاليون للاختبار

```
نحتاج 5-10 أشخاص يمثلون الشرائح المختلفة:

شخص 1: "أحمد — طالب جامعي تقني"
├── 📱 يستخدم جوال سامسونج A (متوسط)
├── 🌐 إنترنت مقبول
├── 👍 توقعات: سهل
└── 🎯 يختبر: السرعة، تجربة المستخدم

شخص 2: "علي — موظف حكومي (50 سنة)"
├── 📱 يستخدم جوال رخيص (Tecno)
├── 🌐 إنترنت ضعيف
├── 👎 توقعات: قد يواجه صعوبة
└── 🎯 يختبر: سهولة الاستخدام، الخطأ الشائع

شخص 3: "محمد — مغترب في السعودية"
├── 📱 يستخدم آيفون
├── 🌐 إنترنت جيد
├── 👍 توقعات: سهل
└── 🎯 يختبر: الدفع من الخارج، تجربة المغترب

شخص 4: "فاطمة — ربة منزل"
├── 📱 تستخدم جوال الزوج (قديم)
├── 🌐 إنترنت ضعيف جداً
├── 👎 توقعات: صعوبة في رفع الصورة
└── 🎯 يختبر: وضوح التعليمات، رفع الصورة بالجوال

شخص 5: "خالد — صاحب مكتب لوتري (منافس)"
├── 💻 يستخدم كمبيوتر + جوال
├── 🌐 إنترنت ممتاز
├── 👍 توقعات: خبير
└── 🎯 يختبر: الاحترافية، الأمان، مقارنة مع خدمته
```

## 3. كيف نختبر؟

### 3.1 البروتوكول (Testing Script)

```
⏰ المدة: 30-45 دقيقة لكل مستخدم
📍 المكان: واتساب فيديو كول (عن بُعد) أو حضور

المرحلة 1: قبل الاختبار (5 دقائق)
├── شرح بسيط: "نحن نختبر موقع جديد للتسجيل في اللوتري"
├── لا تشرح كيفية الاستخدام — نريد رد فعل طبيعي
├── أخبره: "تحدث بصوت عالٍ عن إيش تفكر"
└── وافق على تسجيل الشاشة (للمراجعة)

المرحلة 2: المهام (20 دقيقة)
├── المهمة 1: "افتح الرابط وسجل حساب جديد"
│   └── 📝 ماذا يفعل أولاً؟ هل يقرأ أم يضغط؟
├── المهمة 2: "املأ بياناتك الشخصية"
│   └── 📝 هل يفهم الحقول؟ هل يحتار في شيء؟
├── المهمة 3: "ارفع صورتك"
│   └── 📝 هل يقرأ التعليمات؟ هل ينجح من أول مرة؟
├── المهمة 4: "ادفع 1,000 ريال وارفَع الإيصال"
│   └── 📝 هل يفهم العملية؟ هل يثق في الدفع؟
└── المهمة 5: "اختبر الموقع"
    └── 📝 هل يضغط على الروابط التالفة؟ هل يقرأ الـ FAQ؟

المرحلة 3: بعد الاختبار (10 دقائق)
├── ما أكثر شيء عجبك؟
├── ما أكثر شيء صعب عليك؟
├── هل تثق في الموقع؟ (1-5)
├── هل تدفع 1,000 ريال؟ (نعم/لا/متردد)
└── أي نصيحة لتحسين الموقع؟
```

### 3.2 ماذا نسجّل؟

```
لكل مستخدم:
├── ✅ نجح في المهمة بدون مساعدة
├── ⚠️ نجح مع مساعدة (سؤال)
├── ❌ فشل في المهمة
└── 💡 تعليق أو اقتراح
```

## 4. المهام الحرجة (Critical Paths)

### 4.1 التدفق الأساسي (Happy Path)

```
مهمة حرجة 1: التسجيل
├── يكتب رقم هاتف ← OTP ← يدخل الرمز ← داخل ✅
├── الوقت المتوقع: < 30 ثانية
└── ⚠️ إذا أطول: مشكلة في واجهة OTP

مهمة حرجة 2: رفع الصورة
├── يضغط "رفع صورة" ← يختار صورة ← AI يفحصها ← نتيجة ✅
├── الوقت المتوقع: < 10 ثوانٍ
└── ⚠️ إذا رفضت أول صورة: هل يفهم السبب؟

مهمة حرجة 3: الدفع
├── يختار طريقة ← ينسخ رقم الحساب ← يحول ← يصور ← يرفع ✅
├── الوقت المتوقع: < 2 دقيقة
└── ⚠️ إذا تردد: مشكلة في الثقة
```

### 4.2 التدفقات البديلة (Edge Cases)

```
مهمة 4: صورة مرفوضة من AI
├── كم مرة يحاول؟
├── هل يفهم سبب الرفض؟
└── هل يستسلم؟

مهمة 5: خطأ في الإدخال
├── ماذا إذا كتب رقم جواز خطأ؟
├── هل يقرأ رسالة الخطأ؟
└── هل يعرف كيف يصحح؟

مهمة 6: انقطاع الإنترنت
├── في منتصف التسجيل
├── ماذا يحدث للبيانات؟
└── هل يكمل من حيث توقف؟
```

## 6. الجدول الزمني للاختبار

```
قبل الإطلاق بـ 3 أسابيع:
├── 🏠 اختبار داخلي (فريق التطوير)
│   └── نختبر كل الوظائف (التقني)
│
قبل الإطلاق بـ أسبوعين:
├── 👨‍👩‍👧 اختبار مع 3-5 مستخدمين (الدائرة القريبة)
│   └── أصدقاء، عائلة — نصلح المشاكل الواضحة
│
قبل الإطلاق بـ أسبوع:
├── 🌍 اختبار مع 5-10 مستخدمين حقيقيين
│   └── من الشرائح المختلفة — آخر تعديلات
│
بعد الإطلاق بأسبوع:
├── 📊 تحليل الاستخدام الحقيقي
│   └── هل يوجد اختلاف بين ما اختبرناه والواقع؟
│
بعد الإطلاق بشهر:
├── 🔄 اختبار ثاني مع مستخدمين جدد
└── تحسينات بناءً على الاستخدام الحقيقي
```
