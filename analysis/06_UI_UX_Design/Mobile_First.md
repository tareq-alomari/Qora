## 1. واقع الجوال في اليمن

### 1.1 الإحصاءات

```
┌───────────────────────────────────────────────┐
│ إحصاءات الجوال في اليمن (2025-2026)            │
├───────────────────────────────────────────────┤
│ 📱 نسبة مستخدمي الجوال: ~80%+ من السكان       │
│ 💻 نسبة مستخدمي الكمبيوتر: ~15-20%            │
│ 🌐 الإنترنت عبر الجوال: ~95% من الاتصالات     │
│ 🐌 سرعة الإنترنت المتوسطة: 5-15 Mbps          │
│ 💰 سعر الباقة الشهرية: 3,000-7,000 YR         │
│ 📦 استهلاك البيانات: 1-5 GB/شهر               │
│ 📱 أنظمة التشغيل: Android 70% / iOS 5% / Other│
└───────────────────────────────────────────────┘

العوامل المؤثرة:
├── ⚡ انقطاع الكهرباء (الجوال يشتغل على البطارية)
├── 🌐 ضعف الإنترنت في المناطق الريفية
├── 💸 غياب الحواسيب لدى معظم الأسر
└── 📱 الجوال = الإنترنت الوحيد لكثير من اليمنيين
```

### 1.2 ماذا يعني هذا لـ Qor3a؟

```
يجب أن يكون الموقع:
├── ✅ سطح المكتب ليس أولوية أبداً
├── ✅ الجوال هو أول وآخر شيء
├── ✅ حتى لو الموقع يفتح على كمبيوتر → فكر بالجوال أولاً
└── ✅ لا تعتمد على ميزات سطح المكتب (hover, right-click...)
```

## 2. PWA (Progressive Web App)

### 2.1 لماذا PWA وليس تطبيق؟

```
┌───────────────────────────────────────────────────┐
│  PWA              │  تطبيق Android                  │
├───────────────────┼────────────────────────────────┤
│  مجاني (HTML/CSS)  │  $0 لكن يحتاج نشر في Google Play│
│  لا يحتاج تحميل    │  يحتاج 30-50 MB تحميل           │
│  يشتغل على iOS     │  يحتاج تطبيق منفصل iOS          │
│  تحديث تلقائي      │  تحديث يدوي عبر المتجر           │
│  تقسيم 1-2 يوم     │  نشر في المتجر: 3-7 أيام         │
│  إشعارات Push      │  إشعارات Push                    │
│  يعمل Offline جزئياً│  يعمل Offline أفضل               │
└───────────────────┴────────────────────────────────┘

لماذا PWA يكفي لنا:
├── ✅ لا نريد أن يجبر المستخدم على تحميل تطبيق
├── ✅ المستخدم يزور الموقع و"يضيف للشاشة الرئيسية"
├── ✅ يأخذ 1-2 يوم تطوير (بدلاً من شهر للتطبيق)
├── ✅ يكفي لموسم التسجيل (شهرين)
└── 🟡 تطبيق Android لاحقاً (المرحلة الثانية)
```

### 2.2 ماذا نقدم في PWA

```
عند تثبيت PWA (إضافة للشاشة الرئيسية):
├── 📱 أيقونة على سطح الجوال (مثل التطبيق)
├── 🚀 يفتح بدون متصفح (شاشة كاملة)
├── 🔔 إشعارات Push (حتى لو الموقع مقفول)
├── 💨 سريع جداً (Cached assets)
└── 📡 يعمل جزئياً بدون إنترنت (offline page)

الكود:
├── manifest.json (أيقونة، اسم، ألوان)
├── service-worker.js (كاش، إشعارات)
└── vite.config.js (PWA plugin)
```

## 3. أداء الموقع على الإنترنت الضعيف

### 3.1 حجم الموقع

```
المكون          | الحجم      | استراتيجية التخفيف
────────────────┼────────────┼─────────────────────
HTML (index)    | 5-10 KB    | Static (CDN)
CSS (Tailwind)  | 15-25 KB   | Purged CSS
JS (React)      | 50-80 KB   | Code Splitting
JS (Zustand)    | 5 KB       | Minimal
JS (Router)     | 20 KB      | Lazy Loading
JS (React Hook Form) | 10 KB | Needed
صور (UI)        | 50-100 KB  | WebP + Lazy Loading
صورة العميل     | 200-500 KB | Compressed (JPEG 80%)
AI Check        | ---        | API call (separate)
────────────────┼────────────┼─────────────────────
المجموع الأول   | ~200 KB    | 🟢 ممتاز
بعد التحميل     | ~500 KB    | 🟡 مقبول
```

### 3.2 استراتيجيات التخفيف

```
Images:
├── 🖼️ WebP (أصغر 30% من JPEG)
├── 📱 Lazy Loading (تحميل الصور فقط عندما تظهر)
├── 🔄 Progressive JPEG (يظهر مشوش ثم واضح)
└── 📦 Compress على السيرفر (Sharp.js)

JavaScript:
├── 📦 Code Splitting (كل صفحة تتحمل JS خاص فيها)
├── 🚫 لا مكتبات ضخمة (بدون Three.js, D3, etc.)
├── ❌ لا Font Awesome (استخدم SVG بدلاً منه)
├── ✅ Tailwind CSS (صغير بعد Purge)
└── ✅ Zustand (أصغر من Redux)

Caching:
├── 📂 Service Worker يخزن في الـ Cache:
│   ├── الصفحة الرئيسية (instant load)
│   ├── الشعار والأيقونات
│   ├── الـ CSS (لمدة شهر)
│   └── الـ JS (لمدة أسبوع)
└── 🔄 API calls مخزنة مؤقتاً (60 ثانية)
```

### 3.3 الـ Offline Mode

```
ماذا لو انقطع الإنترنت؟

PWA يمكنه عرض:
├── 🏠 الصفحة الرئيسية (مخزنة)
├── 📱 رسالة: "أنت غير متصل بالإنترنت"
└── 🔄 "سيتم إكمال طلبك تلقائياً عند عودة الاتصال"

في الـ 8-Step Wizard:
├── ⚠️ تحذير في البداية: "تأكد من اتصالك بالإنترنت"
├── كل Step يرسل API call منفصل
├── إذا انقطع في منتصف الإدخال:
│   ├── البيانات محفوظة في localStorage
│   └── يكمل من حيث توقف
└── 💡 لا نحتاج Offline كامل (فقط حفظ مؤقت)
```

## 4. تصميم الجوال أولاً (Mobile-First Design)

### 4.1 Touch Targets

```
في اليمن، معظم المستخدمين يستخدمون:
├── 📱 جوال رخيص (شاشة 5-6 إنش)
├── 👆 أيدي أكبر من المتوسط
└── 🖐️ قد يستخدم الإصبع السبابة

القواعد:
├── ✅ كل الأزرار ≥ 48px (touch target)
├── ✅ تباعد بين الأزرار ≥ 8px
├── ✅ لا أزرار متجاوزة جداً
├── ✅ حقول الإدخال ≥ 44px
└── ❌ ممنوع hover states (لا hover في الجوال)

ما يعمل في الجوال:
┌─────────────────────┐
│ [      سجّل الآن    ]│ ← 48px ارتفاع
├─────────────────────┤
│ ○ ○ ○ ○ ○ ○ ○ ○   │ ← دوائر راديو بحجم مناسب
├─────────────────────┤
│ ┌─────────────────┐ │
│ │ رقم الهاتف      │ │ ← 44px
│ └─────────────────┘ │
└─────────────────────┘
```

### 4.2 تخطيط الشاشة

```
Width: 360-414px (أصغر شاشة مقبولة)
Height: 640-896px

قواعد التخطيط:
├── 📐 Single column (عمود واحد — لا multi-column)
├── 📏 الهامش الجانبي: 16px
├── 📐 كل عنصر يأخذ العرض الكامل
├── 🚫 لا sidebars في الجوال
└── 🚫 لا جداول (tables) — استخدم بطاقات (cards)

القوائم:
├── 📱 القائمة الرئيسية: Bottom Navigation (أسفل الشاشة)
├── 📱 القوائم الثانوية: Drawer (يسار)
└── 📱 الفلاتر: Modal (شاشة منبثقة)

Keyboard:
├── 🖐️ الحقول الرقمية ← type="tel"
├── 📅 التاريخ ← type="date"
└── 📧 الإيميل ← type="email"
```

## 5. استهلاك البيانات

### 5.1 كم يستهلك الموقع؟

```
زيارة الصفحة الرئيسية:
├── أول مرة: ~200 KB
├── مرة ثانية (من cache): ~5 KB
└── ⚡ أسرع مع PWA

الـ 8-Step Wizard (كامل):
├── الخطوة 1 (بيانات): ~20 KB (API)
├── الخطوة 2 (عائلة): ~15 KB
├── الخطوة 3 (صورة): ~300 KB (upload)
├── الخطوة 4 (جواز): ~10 KB
├── الخطوة 5 (عنوان): ~10 KB
├── الخطوة 6 (إيميل): ~5 KB
├── الخطوة 7 (دفع): ~250 KB (إيصال)
└── الخطوة 8 (تأكيد): ~10 KB

المجموع: ~600 KB لكل طلب
┌─────────────────────────────────────────┐
│ مقارنة:                                  │
│ ⚡ فيديو تيك توك واحد: 20-50 MB          │
│ 📱 طلب Qor3a كامل: ~0.6 MB               │
│ 💰 التكلفة: ~3-5 ريال من الباقة           │
└─────────────────────────────────────────┘
```

### 5.2 تقليل استهلاك البيانات

```
├── 🖼️ الصور مضغوطة: 80% جودة (غير محسوسة)
├── 🌐 Lazy Loading: ما نحمّل إلا اللي يظهر
├── 📦 Prefetch: نحمل الخطوة التالية خفية
├── 🔄 واجهة الـ API: ترجع بيانات قليلة (لا junk data)
└── 🚫 لا فيديوهات تلقائية (autoplay)
```

## 6. الـ Device Limitations

### 6.1 مشاكل الأجهزة الرخيصة في اليمن

```
📱 أجهزة شائعة في اليمن:
├── Samsung Galaxy A Series ($100-200)
├── Xiaomi Redmi ($80-150)
├── Tecno/Infinix ($60-120)
└── Huawei القديمة ($50-100)

المواصفات المشتركة:
├── 💾 RAM: 2-4 GB
├── 📱 شاشة: 720p (160-200 DPI)
├── 📷 كاميرا: 8-13 MP (غير ممتازة)
├── ⚡ معالج: MediaTek Helio أو Snapdragon 400
└── 🐌 Android 10-12 (قديم)

ماذا يعني هذا:
├── ✅ CSS Animations → بسيطة (لا hardware acceleration)
├── ✅ Images → منخفضة الدقة (لا 4K)
├── ✅ Memory → لا تترك أشياء في الذاكرة
├── ✅ Camera → جودة الصورة محدودة (يحتاج إضاءة جيدة)
└── ✅ Battery → لا تجهد المعالج (ممنوع polling كل ثانية)
```
