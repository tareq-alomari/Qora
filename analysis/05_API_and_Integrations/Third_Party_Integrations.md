## 1. نظرة عامة على مزودي الدفع في اليمن

### 1.1 السوق اليمني للدفع الإلكتروني

```
اليمن:
├── 💳 80%+ من التعاملات نقداً
├── 📱 40-50% يملكون محفظة إلكترونية (كريمي، جيب...)
├── 🏦 20% فقط لديهم حساب بنكي
└── 🌐 الـ API متاح لبعض المزودين بس بترخيص

مزودو الدفع الرئيسيون:
┌──────────────────────────────────────────────┐
│  اسم المزود  │ النوع     │   الانتشار   │ API │
├──────────────┼───────────┼───────────────┼─────┤
│ كريمي (Kuraimi) │ محفظة   | صنعاء+شمال  | محدود│
│ جيب (Jeeb)     │ محفظة   | اليمن كله    | ✅   │
│ ون كاش (One Cash) │ محفظة | عدن+جنوب   | محدود│
│ موبايل موني     │ محفظة   | المناطق     | ❌   │
│ MTN Money     │ محفظة   | محدود       | ❌   │
│ بنك التسليف    │ تحويل   | مؤسسات      | ❌   │
└──────────────┴───────────┴───────────────┴─────┘
```

## 2. تحليل كل مزود بالتفصيل

### 2.1 كريمي (Kuraimi Exchange)

```
نظرة عامة:
├── 🏢 شركة: كريمي للصرافة
├── 📍 المقر: صنعاء
├── 📱 الخدمة: محفظة إلكترونية + تحويل أموال
├── 👥 المستخدمون: ~2-3 مليون مستخدم (تقدير)
├── 🌍 يمنيون في الخارج: ✅ يقبل حوالات من السعودية
└── ⭐ السمعة: جيدة (موثوق)

المميزات:
├── ✅ الأكثر انتشاراً في صنعاء والشمال
├── ✅ يقبل الإيداع من فروع كثيرة
├── ✅ يدعم التحويل من الخارج (السعودية، الإمارات)
└── ✅ واجهة مستخدم عربية بسيطة

العيوب:
├── ❌ ما فيه API للاستعلام عن المعاملات (حسب آخر معلومة)
├── ❌ limit يومي للسحوبات
├── ❌ قد يتأخر التحويل في أوقات الذروة
└── ❌ ما يدش روابط الدفع (payment links)

الرسوم:
├── تحويل داخلي → مجاني
├── سحب من فرع → 0.5%
├── تحويل من الخارج → 2-3%
└── فتح محفظة → مجاني

توصيتنا:
├── 🟢 نستخدمه كمحفظة أساسية (الأكثر انتشاراً)
├── 🔴 لا نعتمد عليه في API (مافي)
└── 🟡 مراجعة يدوية للإيصالات
```

### 2.2 جيب (Jeeb)

```
نظرة عامة:
├── 🏢 شركة: جيب للتكنولوجيا المالية
├── 📍 المقر: صنعاء + عمان
├── 📱 الخدمة: محفظة إلكترونية + خدمات دفع
├── 👥 المستخدمون: ~1-2 مليون
├── 🌍 يمنيون في الخارج: ✅ يقبل Apple Pay + cards
└── ⭐ السمعة: جيدة جداً (أكثرهم تقدماً)

المميزات:
├── ✅ عنده API للاستعلام عن المعاملات (Payment Gateway)
├── ✅ يدعم Merchant Account (حساب تاجر)
├── ✅ رابط دفع (Payment Link) — يرسل رابط للعميل
├── ✅ يدعم بطاقات فيزا/ماستر كارد
└── ✅ يدعم Apple Pay

العيوب:
├── ❌ الأقل انتشاراً في اليمن (مقارنة بكريمي)
├── ❌ بعض المناطق الريفية ما تعرفه
├── ❌ فروع محدودة (تعتمد على الوكلاء)
└── ❌ شروط فتح حساب تاجر معقدة

الرسوم:
├── تحويل بين محافظ → مجاني
├── سحب إلى بنك → 1%
├── Merchant API → 2.5% لكل معاملة
├── Payment Link → 2% لكل معاملة
└── فتح محفظة → مجاني

توصيتنا:
├── 🟢 أفضل خيار لـ API (المرحلة الثانية)
├── 🟢 Payment Link يسهل التحصيل
├── 🔴 في المرحلة الأولى: يدوي
└── 🟡 المرحلة الثانية: API مباشر
```

### 2.3 ون كاش (One Cash)

```
نظرة عامة:
├── 🏢 شركة: ون كاش للخدمات المالية
├── 📍 المقر: عدن
├── 📱 الخدمة: محفظة إلكترونية + تحويلات
├── 👥 المستخدمون: ~500k-1 مليون (جنوب اليمن)
└── ⭐ السمعة: مقبولة

المميزات:
├── ✅ منتشر في عدن وحضرموت (الجنوب)
├── ✅ يدعم الحوالات من الخارج
├── ✅ رسوم منخفضة
└── ✅ يدعم الإيداع من فروع كثيرة في الجنوب

العيوب:
├── ❌ ما فيه API
├── ❌ محدود جغرافياً (شمال اليمن ما يعرفوه)
├── ❌ مشاكل تقنية أحياناً
└── ❌ خدمة عملاء ضعيفة

الرسوم:
├── تحويل داخلي → مجاني
├── سحب → 0.5%
├── تحويل خارجي → 2-3%
└── فتح محفظة → مجاني

توصيتنا:
├── 🟡 نستخدمه كمحفظة ثانوية (لعملاء الجنوب)
└── 🔴 لا نعتمد عليه في API
```

### 2.4 موبايل موني (Mobile Money Yemen)

```
نظرة عامة:
├── 🏢 شركة: Mobile Money Yemen
├── 📍 الخدمة: محفظة إلكترونية
├── 👥 المستخدمون: ~200-500k (مناطق محدودة)
└── ⭐ السمعة: جديدة، غير معروفة كثيراً

المميزات:
├── ✅ يدعم الإيداع من وكلاء في مناطق محدودة
├── ✅ رسوم منخفضة
└── ✅ سهل الاستخدام

العيوب:
├── ❌ ما فيه API
├── ❌ محدود جداً (مناطق معينة)
├── ❌ ما في خدمة عملاء قوية
└── ❌ غير معروف في اليمن

توصيتنا:
├── 🔴 خيار احتياطي فقط
└── 🔴 لا نعتمد عليه أساساً
```

## 3. استراتيجية الدفع (Payments Strategy)

### 3.1 المرحلة 1: يدوي بالكامل (MVP)

```
┌─────────────────────────────────────────────────────────┐
│                  Payment Flow — MVP                       │
│                                                           │
│  1. العميل يختار طريقة الدفع:                             │
│     ○ كريمي  ● جيب  ○ ون كاش  ○ موبايل موني              │
│                                                           │
│  2. نعرض رقم الحساب حسب المزود المختار:                   │
│     ┌─────────────────────────────────────┐               │
│     │  📱 جيب: 9876543210                  │               │
│     │  الاسم: Qor3a Yemen                  │               │
│     │  المبلغ: 1,000 ريال                   │               │
│     └─────────────────────────────────────┘               │
│                                                           │
│  3. العميل ينسخ الرقم ويحول من تطبيق جيب                   │
│                                                           │
│  4. العميل يصور الإشعار ويرفعه في الموقع                  │
│                                                           │
│  5. ⏳ الموظف يفحص الإيصال يدوياً ويؤكد الدفع              │
│     ← المدة: < 4 ساعات                                    │
└─────────────────────────────────────────────────────────┘

الإيجابيات:
├── ✅ بسيط، لا يحتاج API
├── ✅ ينفع لأي مزود
└── ✅ لا رسوم على Qor3a

السلبيات:
├── ❌ بطيء (4 ساعات)
├── ❌ يدوي (يحتاج موظف)
├── ❌ احتمال تزوير الإيصال
└── ❌ العميل لازم يصور ويرفع
```

### 3.2 المرحلة 2: Payment Link (بعد MVP)

```
ما هو Payment Link؟
├── رابط تشتريه من جيب أو كريمي
├── ترسله للعميل على واتساب
├── العميل يضغط الرابط ويدفع مباشرة
└── أنت تستلم إشعار فوري بالدفع

التدفق:
┌─────────────────────────────────────────┐
│  1. الموظف يضغط "إنشاء رابط دفع"        │
│  2. النظام يرسل للعميل على واتساب:      │
│     "اضغط هنا للدفع: qor3a.com/pay/XXX" │
│  3. العميل يدفع مباشرة (2 دقيقة)        │
│  4. النظام يستلم Webhook (فوري)         │
│  5. ✅ الدفع مؤكد تلقائياً              │
└─────────────────────────────────────────┘

الإيجابيات:
├── ✅ فوري (بدون انتظار 4 ساعات)
├── ✅ آلي (لا يحتاج موظف)
├── ✅ آمن (لا تزوير)
├── ✅ العميل ما يحتاج يصور
└── ✅ العميل ما يحتاج يخرج من الموقع

السلبيات:
├── ❌ رسوم 2-2.5% على كل معاملة
├── ❌ يحتاج حساب تاجر (Merchant Account)
├── ❌ فقط المزودات اللي تدعمه (جيب حالياً)
└── ❌ بعض العملاء لا يثقون في الروابط

الرسوم:
├── 1,000 YR × 2% = 20 YR رسوم
└── صافي القرعة: 980 YR (بدلاً من 1,000)

متى نطبقه؟
├── بعد 3-6 أشهر من الإطلاق
├── بعد ما نبني ثقة العملاء
└── أو نموّن payment links لأي طلب
```

### 3.3 المرحلة 3: API مباشر (مستقبل)

```
التكامل المباشر مع جيب API:
├── العميل يدفع داخل الموقع (ما يخرج لتطبيق جيب)
├── يستلم إشعار QR code
├── يمسحه من تطبيق جيب
└── الفلوس تصل فوراً + Webhook تأكيد

يتطلب:
├── 🏢 حساب Merchant عند جيب
├── 📄 موافقة البنك المركزي اليمني
├── 💻 تكامل API (أيام)
└── 💰 رسوم 2.5% لكل معاملة

هذا أفضل حل — لكنه يحتاج وقت وتراخيص.
```

## 4. مقارنة مزودي الدفع — لأي نعتمد

| المزود | الانتشار | API | Payment Link | الثقة | رسوم | توصيتنا |
|--------|---------|-----|-------------|-------|------|---------|
| 🥇 **جيب** | عالي | ✅ | ✅ | عالية | 2% | **أساسي — الرقم 1** |
| 🥇 **كريمي** | عالي جداً | محدود | ❌ | عالية | 0% | **أساسي — يدوي** |
| 🥉 **ون كاش** | متوسط | ❌ | ❌ | متوسطة | 0% | **احتياطي (جنوب)** |
| ❌ **موبايل موني** | محدود | ❌ | ❌ | محدودة | 0% | **لا نعتمد عليه** |

### التوصية النهائية

```
في الإطلاق:
├── ✅ نعرض: كريمي + جيب + ون كاش
├── ✅ العميل يحول يدوياً
├── ✅ الموظف يؤكد الإيصال
└── 🎯 نركز على جيب لاحقاً (Payment Link)

في المرحلة الثانية (3-6 شهور):
├── ✅ نضيف Payment Link من جيب
├── ✅ العميل يدفع برابط ولا يحتاج يصور
└── ✅ نبدأ نشتغل على Merchant API

في المرحلة الثالثة (سنة):
├── ✅ API مباشر مع جيب
├── ✅ الدفع داخل الموقع
└── ✅ إشعارات فورية
```
