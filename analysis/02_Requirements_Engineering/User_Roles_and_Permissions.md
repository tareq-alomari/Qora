# 👥 صلاحيات المستخدمين — User Roles & Permissions (RBAC)

> **Qor3a (قرعة)** — الأدوار، الصلاحيات، مصفوفة الوصول

---

## 1. الأدوار (Roles)

| الدور | الكود | الوصف | العدد المتوقع |
|-------|-------|-------|--------------|
| **عميل** | `client` | المسجل في الخدمة — يرى طلباته فقط | 100-3,000 |
| **موظف** | `employee` | يراجع الطلبات ويؤكد الدفع ويدخل البيانات رسمياً | 2-5 |
| **مدير** | `admin` | يدير النظام والمستخدمين والتقارير والإعدادات | 1-2 |

## 3. هيكل الـ Middleware

```javascript
// 1. Auth — يفك JWT ويضبط req.user
router.use(authMiddleware)

// 2. Role Check
router.get('/orders', requireRole('client', 'employee', 'admin'), controller.list)
router.post('/admin/users', requireRole('admin'), controller.createUser)

// 3. Ownership Check (العميل يرى طلباته فقط)
router.get('/orders/:id', requireOwnership, controller.getById)

// 4. Permission Check (للموارد الحساسة)
router.patch('/orders/:id/status', requirePermission('change_status'), controller.updateStatus)
```

### Ownership Middleware Logic

```javascript
const requireOwnership = (req, res, next) => {
  if (req.user.role === 'admin') return next()
  if (req.user.role === 'employee') return next()

  // Client: only their own resources
  const resource = await orderModel.findById(req.params.id)
  if (resource.user_id !== req.user.id) {
    throw new AppError(403, 'FORBIDDEN', 'ليس لديك صلاحية لهذا الطلب')
  }
  next()
}
```

## 4. صلاحيات خاصة (بحسب حالة الطلب)

| الحالة | من يمكنه التعديل |
|--------|-----------------|
| Draft | العميل فقط |
| Data_Entry_Complete | العميل (قبل رفع الصورة) |
| Photo_Pending | العميل (رفع/إعادة الصورة) |
| Photo_Rejected | العميل (إعادة المحاولة) |
| Photo_Accepted | النظام (تلقائي) |
| Payment_Pending | العميل (رفع الإيصال) |
| Payment_Verification | الموظف + المدير |
| Needs_Correction | العميل (حسب ملاحظة الموظف) |
| Approved | الموظف (تقديم للموقع) |
| Submitted | الموظف + المدير |
| Completed | قراءة فقط للجميع |
| Cancelled | قراءة فقط للجميع |

## 5. مبدأ最小 الصلاحية (Least Privilege)

```
كل دور يملك فقط الصلاحيات التي يحتاجها لأداء مهامه:
├── العميل: طلباته + بياناته + مدفوعاته فقط
├── الموظف: كل الطلبات (قراءة) + تغيير حالة + تأكيد دفع + تقديم
└── المدير: كل شيء + إدارة المستخدمين + الإعدادات + التقارير
```

> لا يوجد دور "مشرف" (Super Admin) منفصل — المدير هو أعلى صلاحية.
> يمكن إضافة صلاحيات دقيقة مستقبلاً (مثلاً: موظف تأكيد دفع فقط).
