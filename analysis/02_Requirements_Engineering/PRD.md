# 📋 Product Requirements Document — PRD

> **Qor3a (قرعة)** — منصة التسجيل الآلي في DV Lottery  
> **النسخة**: V1.0 | **الحالة**: موافقة | **آخر تحديث**: يوليو 2026

---

## 1. Product Vision

> **"منصة رقمية تجعل التسجيل في DV Lottery سهلاً، سريعاً، وموثوقاً لكل يمني — بنصف سعر السوق وضعف الخدمة."**

نبدأ كمنصة متخصصة في DV Lottery ونتوسع إلى بوابة خدمات إلكترونية شاملة (تأشيرات، ترجمة، جوازات).

---

## 2. Target Audience

| الشريحة | العمر | عدد المستخدمين المقدر | الاحتياج |
|----------|-------|----------------------|----------|
| الشباب اليمني | 18-35 | 5M+ | تسجيل سريع من الجوال |
| المغتربون اليمنيون | 25-50 | 2M+ | لا يوجد مكتب في بلد الاغتراب |
| الأسر اليمنية | 25-55 | 1M+ | تسجيل أفراد الأسرة |
| المكاتب (B2B لاحقاً) | — | 50-100 | أتمتة تسجيلات زبائنهم |

---

## 3. Problem Statement

| المشكلة | التأثير |
|---------|---------|
| المكاتب التقليدية تاخذ 2,000 YR | غالية - غير متاحة للجميع |
| دوام المكاتب محدود (8-5) | يصعب الوصول بعد الدوام |
| أخطاء في إدخال البيانات | رفض الطلب من وزارة الخارجية |
| صور مرفوضة (لا أحد يراجع) | إضاعة فرصة السنة |
| لا متابعة بعد التسجيل | ما تعرف إذا فزت ولا لا |

---

## 4. Solution Overview

```
قنوات المستخدم
┌───────────────────────────────────────────────────────┐
│                   Mobile Web (PWA)                      │
│              WhatsApp (إشعارات + دعم)                    │
│              SMS (طوارئ)                               │
└──────────────────┬────────────────────────────────────┘
                   │
┌──────────────────▼────────────────────────────────────┐
│              8-Step Registration Wizard                 │
│                                                        │
│ 1. تسجيل ← 2. بيانات ← 3. تصوير ← 4. AI ← 5. دفع     │
│ 6. مراجعة ← 7. تسليم رقم التأكيد ← 8. فحص النتيجة      │
└────────────────────────────────────────────────────────┘
```

**مدة التسجيل**: < 5 دقائق  
**المعالجة**: من 1-24 ساعة (حسب وقت تأكيد الدفع)  
**الفحص (مايو)**: 3-5 دقائق لكل طلب

## 5. Functional Requirements (FR)

### 5.1 Auth Module (FR-01)
| ID | المتطلب | أولوية | ملاحظات |
|----|---------|--------|---------|
| FR-01.1 | تسجيل برقم الهاتف + OTP | عالية | لا كلمة مرور |
| FR-01.2 | تسجيل بالبريد الإلكتروني | متوسطة | مع التحقق |
| FR-01.3 | OAuth Google/Apple | منخفضة | المرحلة 2 |
| FR-01.4 | JWT Access (24h) + Refresh (7d) | عالية | Refresh في httpOnly cookie |
| FR-01.5 | استعادة حساب عبر OTP للهاتف | عالية | |

### 5.2 Registration Wizard (FR-02)
| ID | المتطلب | أولوية |
|----|---------|--------|
| FR-02.1 | حفظ تلقائي كل 10 ثوانٍ (Auto-Save) | عالية |
| FR-02.2 | إدخال البيانات خطوة بخطوة (5 خطوات) | عالية |
| FR-02.3 | إدخال بيانات الزوج/الزوجة ديناميكياً | عالية |
| FR-02.4 | إدخال بيانات الأبناء بطريقة مكررة | عالية |
| FR-02.5 | التحقق من صحة كل حقل (Validation) | عالية |
| FR-02.6 | حفظ كمسودة (Draft) + العودة لاحقاً | عالية |
| FR-02.7 | مراجعة وملخص قبل الإرسال النهائي | عالية |

### 5.3 AI Photo Validation (FR-03)
| ID | المتطلب | أولوية |
|----|---------|--------|
| FR-03.1 | التقاط صورة من الكاميرا في المتصفح | عالية |
| FR-03.2 | إطار إرشادي لوضع الوجه | عالية |
| FR-03.3 | **ممنوع قطعاً**: تعديل البكسلات أو استبدال الخلفية | حرجة |
| FR-03.4 | التحقق من: مركز الرأس، حجم الرأس، خلفية بيضاء، إضاءة، وضوح | عالية |
| FR-03.5 | عرض نتيجة الـ AI مع سبب الرفض | عالية |
| FR-03.6 | رفع صورة من المعرض (بديل) | متوسطة |
| FR-03.7 | تخزين نتيجة التحقق في `photo_validation` (JSONB) | عالية |

### 5.4 Payment (FR-04) — إيداع يدوي لا API
| ID | المتطلب | أولوية |
|----|---------|--------|
| FR-04.1 | عرض خيارات الدفع (كريمي، جيب، ون كاش، موبايل موني) | عالية |
| FR-04.2 | عرض رقم حساب قرعة حسب الطريقة المختارة | عالية |
| FR-04.3 | رفع صورة إشعار الحوالة أو إدخال رقم الحوالة | عالية |
| FR-04.4 | `payment_pending` ← `payment_verification` بعد الرفع | عالية |
| FR-04.5 | موظف يؤكد الدفع يدوياً عبر لوحة التحكم | عالية |
| FR-04.6 | OCR لاستخراج رقم الحوالة (المرحلة 2) | متوسطة |

## 7. Constraints

| ID | القيد | السبب |
|----|-------|-------|
| C-01 | **ممنوع تعديل الصورة** | قانون وزارة الخارجية الأمريكية |
| C-02 | CAPTCHA في الموقع الرسمي | الفحص الآلي يحتاج 2Captcha + يدوي |
| C-03 | لا API للدفع | طرق الدفع اليمنية لا تدعم APIs |
| C-04 | VPN/Proxy مطلوب | اليمن قد تكون محظورة من الموقع الرسمي |
| C-05 | الخدمة مساعدة وليست انتحال | يجب توضيح أن الخدمة غير رسمية |

## 8. User Stories Summary

```
US-01: بصفتي عميلاً، أسجل بسرعة بهاتفي فقط
US-02: بصفتي عميلاً، أدخل بياناتي خطوة بخطوة
US-03: بصفتي عميلاً، بياناتي تحفظ تلقائياً
US-04: بصفتي عميلاً، أصور وجهي بإرشادات مباشرة
US-05: بصفتي عميلاً، أعرف فوراً إذا صورتي مقبولة
US-06: بصفتي عميلاً، أدفع بحوالة محلية
US-07: بصفتي عميلاً، أرفع إيصال الدفع من جوالي
US-08: بصفتي عميلاً، أستلم رقم التأكيد فوراً
US-09: بصفتي عميلاً، أفحص نتيجتي بنقرة واحدة
US-10: بصفتي موظفاً، أرى الطلبات حسب الأولوية
US-11: بصفتي موظفاً، أراجع كل شيء في شاشة واحدة
US-12: بصفتي مديراً، أرى إحصائيات فورية
```
