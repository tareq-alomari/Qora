# الصيانة الدورية - Maintenance Schedule

## Qor3a — ماذا نفعل في غير موسم التسجيل؟

---

## 1. أنواع الصيانة

| النوع | التكرار | المدة | الهدف |
|-------|---------|-------|-------|
| 🟢 **يومية** | كل يوم | 10 دقائق | التحقق من الصحة |
| 🟡 **أسبوعية** | كل أسبوع | 30 دقيقة | صيانة روتينية |
| 🟠 **شهرية** | كل شهر | 2-4 ساعات | صيانة عميقة |
| 🔴 **طارئة** | عند اللزوم | غير محدد | إصلاح عطل |
| 🎯 **موسمية** | قبل الموسم | يوم كامل | تجهيز للموسم |

## 2. جدول الصيانة

### 2.1 صيانة يومية (عند بداية الدوام)

```
⏰ المدة: 10 دقائق
👤 المسؤول: الموظف أو المطور

قائمة التحقق:
□ هل الموقع يفتح؟
  → qor3a.com يعود 200
□ هل API شغال؟
  → /api/v1/health يعود 200
□ هل AI Service شغال؟
  → /api/ai/health يعود 200
□ هل Redis شغال؟
  → Queue تعمل
□ هل MinIO شغال؟
  → الصور ترفع وتنزل
□ هل آخر Backup تم أمس؟
  → التحقق من ملف الـ Backup
□ هل توجد أخطاء في Logs الأمس؟
  → مراجعة سريعة

إذا أي شيء لا يعمل:
├── 🟢 مشكلة بسيطة → تصليح فوري
└── 🔴 مشكلة كبيرة → تفعيل خطة الطوارئ
```

### 2.2 صيانة أسبوعية (كل جمعة — دوام مخفف)

```
⏰ المدة: 30 دقيقة
👤 المسؤول: المطور

قائمة التحقق:
□ تحديث الـ Dependencies:
  → npm outdated (هل توجد تحديثات أمنية؟)
  → تحديث فقط الـ patch/minor (لا major)
□ مراجعة الـ Logs:
  → أي أخطاء متكررة؟
  → أي محاولات اختراق؟
□ التحقق من التخزين:
  → df -h (هل أقل من 80%؟)
□ مراجعة الـ Errors:
  → أي خطأ يتكرر > 5 مرات؟
□ SSL:
  → هل الشهادة سارية؟
□ تحليل الاستخدام:
  → عدد الزوار
  → معدل الخطأ
  → أبطأ صفحة
□ الـ Backup:
  → اختبار استرجاع سريع (اختياري — كل شهر)
```

### 2.3 صيانة شهرية (أول السبت من كل شهر)

```
⏰ المدة: 2-4 ساعات
👤 المسؤول: المطور + المدير

ما قبل الصيانة:
├── إشعار للعملاء: "الموقع تحت الصيانة لمدة 2-4 ساعات"
└── التأكد من وجود Backup حديث

قائمة الصيانة:
□ تحديثات أمنية:
  → كل الـ dependencies
  → نظام التشغيل (apt update && apt upgrade)
  → Docker images (docker pull)
□ تنظيف:
  → حذف الصور المؤقتة (أقدم من 60 يوم)
  → Log Rotation (ضغط + حذف القديم)
  → حذف الجلسات المنتهية
  → Vacuum PostgreSQL
□ تحسين الأداء:
  → مراجعة استعلامات SQL البطيئة
  → إضافة Indexes (إن لزم)
  → تحليل Page Speed (Google Lighthouse)
□ مراجعة الأمان:
  → تغيير كلمات السر (اختياري — كل 3 أشهر)
  → مراجعة Audit Logs (أي إجراء مشبوه)
  → التحقق من صلاحية API Keys
□ Backup:
  → Backup كامل (DB + images + config)
  → اختبار استرجاع (اختياري)

ما بعد الصيانة:
├── اختبار سريع: هل كل شيء يعمل؟
└── إشعار: "انتهت الصيانة، كل الخدمات شغالة ✅"
```

### 2.4 صيانة موسمية (قبل موسم التسجيل — أغسطس)

```
⏰ المدة: يوم كامل (8 ساعات)
👤 المسؤول: المطور + المدير + الموظف

قائمة التحضير للموسم:

🕐 الصباح (4 ساعات):
□ توسيع السيرفر (Hetzner → upgrade)
□ اختبار تحميل (Load Testing):
  → 500 مستخدم وهمي في وقت واحد
  → هل API يتحمل؟
  → هل AI يتحمل؟
  → هل الـ Queue يعمل؟
□ تحديث الـ Dependencies كاملة
□ مراجعة جميع API endpoints
□ اختبار الـ State Machine (كل انتقال)

🕐 الظهر (2 ساعة):
□ تجديد حسابات الدفع (كريمي، جيب — شحن الرصيد)
□ شحن 2Captcha API (إن لزم)
□ تجديد Proxy/VPN (للتسجيل الرسمي)
□ اختبار الـ Headless Browser (مع CAPTCHA)

🕐 المساء (2 ساعة):
□ مراجعة المحتوى (FAQ، photo guide)
□ تحديث الأسعار (إن تغيرت)
□ اختبار شامل (full E2E test)
□ Back up كامل

النتيجة:
├── 🟢 كل شيء جاهز للإطلاق
└── ❌ مشكلة → حل قبل الإطلاق
```

## 3. قائمة المشاكل المتوقعة (Playbook)

### 3.1 مشاكل شائعة وحلولها

```
🚨 502 Bad Gateway
├── السبب: Nginx لا يصل إلى API
├── الحل: docker compose restart backend-api
└── وقت الحل: 1-2 دقيقة

🚨 PostgreSQL connection refused
├── السبب: DB مشتغلة أو وصلت للحد الأقصى
├── الحل: docker compose restart postgres
│   أو: زيادة max_connections
└── وقت الحل: 2-5 دقائق

🚨 الصور لا ترفع
├── السبب: MinIO معطل أو مساحة ممتلئة
├── الحل: docker compose restart minio
│   أو: تنظيف الصور القديمة
└── وقت الحل: 5-10 دقائق

🚨 AI Service بطيء
├── السبب: تحميل عالي أو Out of Memory
├── الحل: إعادة تشغيل الخدمة
│   أو: زيادة الـ Resources
└── وقت الحل: 2-5 دقائق

🚨 WhatsApp API لا يرسل
├── السبب: انتهت الـ 1,000 رسالة مجانية
├── الحل: شحن الرصيد في Meta
└── وقت الحل: 10-30 دقيقة

🚨 CAPTCHA لا يحل
├── السبب: انتهى رصيد 2Captcha
├── الحل: شحن الرصيد
│   أو: تفعيل الطابور اليدوي
└── وقت الحل: 5-15 دقيقة
```

### 3.2 ما لا نلمسه في الصيانة (Hard Bans)

```
في الصيانة الدورية، لا نغيّر:
├── ❌ State Machine (إلا إذا ضرورة قصوى)
├── ❌ Schema قاعدة البيانات (migration منفصل)
├── ❌ JWT Secret (إلا إذا اختراق)
├── ❌ API Keys (إلا إذا انتهت)
└── ❌ كود الإنتاج مباشرة (دائماً عن طريق Git)
```

## 4. قائمة التحقق الشهرية (Checklist للإطمئنان)

```
□ الـ Backup: آخر نسخة سليمة؟
□ الـ Logs: أخطاء متكررة؟
□ التخزين: تحت 80%؟
□ الـ Memory: تحت 80%؟
□ الـ CPU: تحت 70% (تحميل عادي)؟
□ SSL: ساري لأكثر من 30 يوم؟
□ Domain: مدفوع لأكثر من 30 يوم؟
□ 2Captcha: رصيد كافي؟
□ واتساب API: رصيد كافي؟
□ حسابات الدفع: شغالة؟
□ VPN/Proxy: شغال؟
□ AI Service: الدقة > 90%؟
□ Page Speed: > 70 (جوال)؟
□ جميع الاختبارات: تمر؟
```
