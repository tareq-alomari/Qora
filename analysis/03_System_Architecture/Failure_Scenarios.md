# سيناريوهات إخفاق الموقع الرسمي — Official Site Failure Scenarios

## Qor3a — ماذا لو travel.state.gov غير القواعد فجأة؟

---

## 1. طبيعة الاعتماد على جهة خارجية

```
Qor3a تعتمد كلياً على موقع وزارة الخارجية الأمريكية:
├── 🌐 travel.state.gov → للتسجيل الرسمي
├── 🌐 dvprogram.state.gov → للفحص
└── 🔗 هذه المواقع خارج سيطرتنا تماماً

المشكلة:
├── ❌ لا API رسمي (نحن نعمل عبر الواجهة فقط)
├── ❌ الموقع قد يتغير بدون إشعار
├── ❌ قد يحظر IP اليمن
└── ❌ قد يقررون إلغاء القرعة أو تغيير نظامها
```

---

## 2. سيناريوهات الفشل (مصنفة حسب الاحتمال)

### 2.1 🟡 سيناريوهات محتملة (قد تحدث)

#### السيناريو A: تغيير النموذج (Form Layout Changed)

```
ماذا يحدث:
├── الموقع الرسمي يغير ترتيب الحقول
├── يضيف/يحذف بعض الأسئلة
├── يغير الـ IDs والفئات (classes)
└── الـ Headless Browser لا يتعرف على الحقول

الاحتمال: متوسط (يحدث كل بضع سنوات)
التأثير: عالي

كيف نكتشف:
├── الـ Headless Browser يفشل في العثور على حقل
├── Error Log: "Element not found: #first-name"
├── الـ Queue يتوقف
└── الموظف يلاحظ: "الموقع تغير!"

الحل:
├── ⏱️ فوري: التبديل إلى الإدخال اليدوي
├── 🔧 خلال ساعات: تحديث الـ Selectors في الكود
├── ✅ اختبار الـ Selectors الجديدة
└── 🔄 إعادة تشغيل الـ Queue

وقت الحل: 2-8 ساعات
الأضرار: تأخير في التسجيل (ليس فشل كامل)
```

#### السيناريو B: CAPTCHA جديد (New CAPTCHA Type)

```
ماذا يحدث:
├── الموقع يغير من reCAPTCHA v2 إلى hCaptcha
├── أو يضيف اختبار صور (Image CAPTCHA)
└── 2Captcha لا يدعم النوع الجديد بعد

الاحتمال: متوسط
التأثير: متوسط

الحل:
├── ⏱️ فوري: تفعيل الطابور اليدوي
├── 🔧 سريع: التحقق من دعم 2Captcha للنوع الجديد
│   (عادةً 2Captcha يدعم خلال أيام)
├── 🔄 إضافة CAPTCHA Solver احتياطي (DeathByCaptcha)
└── 📋 يدوي: الموظف يحل CAPTCHA مباشرة

وقت الحل: 1-48 ساعة
الأضرار: بطء في التسجيل (لكن ليس توقفاً كاملاً)
```

#### السيناريو C: الموقع Down (تحت الصيانة)

```
ماذا يحدث:
├── الموقع الرسمي يعطل (صيانة، مشكلة تقنية)
├── يعرض 503 أو صفحة صيانة
└── لا يمكن الدخول لإكمال التسجيل

الاحتمال: عالي (في الموعد النهائي، الموقع يعلق)
التأثير: عالي

الحل:
├── ⏱️ فوري: إيقاف الـ Queue مؤقتاً
├── 🔄 إعادة المحاولة كل 30 دقيقة
├── 📋 يدوي: الموظف يحاول من VPN/Proxy مختلف
├── ⚠️ إشعار للعملاء: "الموقع الرسمي تحت الصيانة"
└── 🔄 استئناف التشغيل تلقائياً عند عودة الموقع

وقت الحل: غير معروف (يعتمد على وزارة الخارجية)
الأضرار: تأخير (لكن TTL يمدد أحياناً)
```

#### السيناريو D: حظر IP اليمن (Yemen IP Blocked)

```
ماذا يحدث:
├── الموقع الرسمي يحظر كل الـ IPs اليمنية
├── حتى Proxies/VPN قد لا تشتغل
└── لا نستطيع التسجيل

الاحتمال: منخفض-متوسط
التأثير: عالي جداً

الحل:
├── 🔄 تغيير الـ Proxy إلى IP غير يمني (US, Turkey, UAE)
├── 🔄 استخدام VPN (NordVPN, ExpressVPN) — الاشتراك جاهز
├── 🔄 محاولة من سيرفرات في أوروبا
└── ⚠️ إشعار للعملاء: "تأخير بسبب حجب الموقع"

الخطط المتعددة (Redundancy):
├── 🥇 Proxy رئيسي: US IP (DigitalOcean)
├── 🥈 Proxy احتياطي: Turkey IP (Hetzner)
├── 🥉 Proxy احتياطي: UAE IP
└── 📋 يدوي: موظف في السعودية/الأردن يدخل مباشرة

وقت الحل: 10-30 دقيقة (تبديل Proxy)
الأضرار: محدودة (إذا كان عندنا Proxies احتياطية)
```

### 2.2 🔴 سيناريوهات نادرة (قد لا تحدث لكن كارثية)

#### السيناريو E: إلغاء القرعة (DV Lottery Cancelled)

```
ماذا يحدث:
├── الحكومة الأمريكية تقرر إلغاء DV Lottery بالكامل
├── أو تعليقه لسنة
├── أو تغيير النظام بالكامل (نقاط، فيزا جديدة)
└── نموذج عمل Qor3a يموت بين ليلة وضحاها

الاحتمال: نادر جداً (منذ 1990، لم يحدث)
التأثير: كارثي (Qor3a تغلق)

الحل:
├── لا يوجد حل سحري
├── 🚨 استرداد كل العملاء (1,000 × 2,000 = 2,000,000 YR)
├── 🔄 تحول سريع إلى خدمات أخرى:
│   ├── 📄 ترجمة المستندات (فيزا، جواز، شهادات)
│   ├── 📋 استشارات هجرة
│   ├── 🛂 تجديد جواز السفر
│   └── 🏛️ خدمات حكومية إلكترونية
└── 🤝 التواصل مع العملاء: "نحن لم ننتهِ"

وهذا بالضبط سبب أن Qor3a يجب أن تتوسع
من "خدمة لوتري" إلى "منصة خدمات" عموماً
```

#### السيناريو F: الموقع يطلب دفع مباشر (Application Fee)

```
ماذا يحدث:
├── وزارة الخارجية تقرر أن التسجيل لم يعد مجانياً
├── يطلبون $330 (رسوم التأشيرة) عند التقديم
├── عملاؤنا يتوقعون منا دفعها
└── هذا يغير نموذج العمل بالكامل

الاحتمال: نادر جداً (التسجيل مجاني منذ 1990)
التأثير: كبير

الحل:
├── 📢 توضيح: "Qor3a تقدم المساعدة فقط، رسوم الحكومة على العميل"
├── 💰 تغيير النموذج: Qor3a تدفع بالنيابة عن العميل (وتأخذ أجراً)
├── 🔄 إضافة خدمة: "نحن ندفع رسوم التقديم نيابة عنك"
└── 📊 زيادة السعر إلى 1,500-2,000 YR (لتغطية الرسوم)

إذا صار هذا، سوق اليمن سينهار:
├── 1,000 YR مقبول لخدمة مساعدة
├── 1,000 + $330 = 400,000 YR → معظم اليمنيين لا يستطيعون
└── ← هنا نموذج العمل يضعف
```

## 3. نظام الإنذار المبكر (Early Warning System)

### 3.1 ماذا نراقب يومياً؟

```
نحن نراقب الموقع الرسمي حتى خارج الموسم:

يومياً (آلي — Headless Browser Check):
├── ✅ هل الموقع يفتح؟ (HTTP 200)
├── ✅ هل شهادات SSL سارية؟
├── ✅ هل النموذج تغير؟ (هاش الـ HTML)
├── ✅ هل ظهر CAPTCHA جديد؟
└── ✅ هل وقت الاستجابة طبيعي؟ (< 5 ثوانٍ)

أسبوعياً (يدوي — موظف):
├── ✅ فتح الموقع في المتصفح
├── ✅ التحقق من عدم وجود إعلانات تغيير
├── ✅ قراءة آخر الأخبار عن DV Lottery
└── ✅ التأكد من أن Proxy/VPN يعمل

إذا أي شاذ:
├── 🟡 تغيير بسيط → تسجيل + متابعة
├── 🔴 تغيير كبير → تنبيه فوري للمدير
└── 🚨 إنذار → تفعيل خطة الطوارئ
```

### 3.2 قنوات متابعة الأخبار

```
بالنسبة لأخبار DV Lottery، نتابع:
├── 📡 الموقع الرسمي: travel.state.gov
├── 📡 منتديات DV Lottery (USA, UK)
├── 📡 مجموعات Reddit: r/dvlottery
├── 📡 إيميل إشعارات وزارة الخارجية
└── 📡 مجموعات اليمنيين المتقدمين

لو تغيرت القوانين:
├── ⏰ نعلم قبل أي مكتب يمني بأيام
└── 🏆 ميزة تنافسية: نكون أول من يستعد
```

## 4. خطة الطوارئ في 3 دقائق

```
ماذا يفعل الموظف إذا الموقع الرسمي وقع؟

دقيقة 1: التشخيص
├── افتح الموقع الرسمي في المتصفح (يدوي)
├── هل هو أنت ولا الموقع؟
├── اختبر من Proxy مختلف
└── اختبر من VPN مختلف

دقيقة 2: القرار
├── 🟢 الموقع شغال → مشكلة من Proxy → غيّر الـ Proxy
├── 🟡 الموقع بطيء → انتظر 5 دقائق وحاول مرة أخرى
├── 🔴 الموقع معطل → أبلغ المدير
└── 🚨 النموذج تغير → أبلغ المطور فوراً (لا تحاول الإدخال اليدوي)

دقيقة 3: الإجراء
├── وقف الـ Headless Queue
├── إشعار للمدير (واتساب + إيميل + اتصال)
├── إشعار للعملاء (اختياري — إذا التأخير متوقع)
└── سجّل المشكلة في الـ Logs
```

## 5. خطة الـ Backup (ماذا لو فشل كل شيء)

### 5.1 يدوي بالكامل (Manual Mode)

```
إذا الموقع الرسمي غير متوقع وغير قابل للأتمتة:
├── 🖐️ الموظف يدخل البيانات يدوياً (زي المكاتب)
├── ⏰ ~10 دقائق لكل طلب (بدلاً من 3-5)
├── 👨‍💼 نزيد موظفين إضافيين (مؤقتاً)
└── ⏳ يقل عدد الطلبات في اليوم (لكن يستمر)

التكلفة:
├── دوام إضافي للموظف: +50,000 YR/شهر
├── زمن أطول: 10 دقائق/طلب × 50 طلب = 8 ساعات
└── ← ما زلنا نربح (الهامش 800 YR لكل طلب)
```

### 5.2 استئجار مكتب في الخارج (Office Abroad)

```
إذا اليمن محظور بالكامل (IP + Proxy + VPN):
├── 🏢 استئجار شخص في السعودية/الأردن/تركيا
├── 💻 يعطيه الموظف بيانات الطلب
├── 🌐 يسجل من هناك
└── 📨 يرسل Confirmation#

التكلفة:
├── $200-300/شهر لشخص في تركيا
└── أو: 50,000 YR/شهر (راتب عامل حر)

أو: اتفاقية مع مكتب صغير في عمّان أو إسطنبول
```

### 5.3 خدمة التسجيل الخارجية (Backup Agent)

```
الاتفاق مع شخص في الولايات المتحدة:
├── 🇺🇸 أمريكي أو مقيم في أمريكا
├── 💻 يدخل من IP أمريكي (طبيعي 100%)
├── 📋 نرسل له بيانات الطلب
└── 🔄 يستلم Confirmation# ويرجعه

التكلفة: $5-10 لكل طلب
└── (مكلف لكن أفضل من إلغاء الطلب)
```
