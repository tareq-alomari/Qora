# 💾 متطلبات البيانات — Data Requirements

> **Qor3a (قرعة)** — كل البيانات في النظام: هيكل، أحجام، حساسية، تخزين، عمر

---

## 1. تصنيف البيانات حسب الحساسية

```
┌─────────────────────────────────────────────────────────────┐
│ المستوى    │ أمثلة                    │ التخزين             │
├────────────┼──────────────────────────┼──────────────────────┤
│ 🔴 حرجة    │ الصورة الشخصية، الجواز   │ AES-256 + S3 مقفل   │
│ 🟡 حساسة   │ الاسم، الهاتف، العنوان   │ pgcrypto + RBAC     │
│ 🟢 داخلية   │ حالة الطلب، رقم التأكيد  │ PostgreSQL عادي     │
│ ⚪ عامة     │ الأسعار، الخدمات         │ أي مكان             │
└─────────────────────────────────────────────────────────────┘
```

## 2. كيانات البيانات (6 جداول)

| الجدول | البيانات | الحجم المتوقع (السنة 1) | حساسية |
|--------|---------|------------------------|--------|
| **users** | حساب المستخدم (دور، هاتف، بريد) | 1K-5K صف | 🟡 |
| **orders** | الطلب (حالة، سعر، metadata) | 1K-20K صف | 🟢 |
| **applicant_data** | بيانات المتقدم (اسم، صورة، مؤهلات) | 1K-20K صف | 🔴 |
| **payments** | الدفع (مبلغ، إيصال، حالة) | 1K-20K صف | 🟡 |
| **notifications** | الإشعارات (نوع، حالة، وقت) | 50K-500K صف | 🟢 |
| **audit_logs** | سجل التدقيق (إجراء، من/إلى) | 50K-500K صف | 🟢 |

## 3. البيانات غير المهيكلة (JSONB)

| الجدول | العمود | المحتوى | مثال |
|--------|--------|---------|------|
| `applicant_data` | `spouse_data` | بيانات الزوج/الزوجة | `{name, birth_date, nationality}` |
| `applicant_data` | `children_data` | مصفوفة بيانات الأبناء | `[{name, birth_date}, ...]` |

| `audit_logs` | `metadata` | تفاصيل التغيير | `{reason, changed_fields}` |

| العنصر | السنة 1 | السنة 2 | السنة 3 |
|--------|---------|---------|---------|
| PostgreSQL | 500MB | 2GB | 5GB |
| الصور (MinIO) | 2GB | 8GB | 20GB |
| الإيصالات | 500MB | 2GB | 5GB |
| Backups | 5GB | 20GB | 50GB |
| **المجموع** | **~8GB** | **~32GB** | **~80GB** |

---

## 6. قواعد البيانات العامة

| القاعدة | التفاصيل |
|---------|---------|
| **Primary Keys** | UUID v4 (وليس auto-increment) |
| **Timestamps** | `TIMESTAMPTZ` مع `DEFAULT NOW()` |
| **حذف** | Soft delete (`is_active = false`) — لا `DELETE FROM` |
| **التدوين** | `created_at` + `updated_at` لكل جدول |
| **Indexes** | على `user_id`، `status`، `created_at`، `order_id` |
| **التشفير** | pgcrypto للبيانات الحساسة، AES-256 للملفات |
| **النسخ الاحتياطي** | pg_dump يومياً + rsync للصور أسبوعياً |
