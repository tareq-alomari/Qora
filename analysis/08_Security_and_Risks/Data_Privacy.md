## 2. الخصوصية وحماية البيانات (Privacy & Data Protection)

### 2.1 ما هي البيانات الحساسة التي نخزنها؟

```
🔴 شديد الحساسية:
├── 📸 الصورة الشخصية (صورة العميل)
├── 🆔 رقم الجواز
├── 📅 تاريخ الميلاد
└── 👤 الاسم الكامل (أربعة أسماء)

🟡 حساسية متوسطة:
├── 📞 رقم الهاتف
├── 📧 الإيميل
├── 🏠 العنوان
└── 👨‍👩‍👧 بيانات العائلة (زوج/زوجة/أطفال)

🟢 غير حساس:
├── حالة الطلب
├── تاريخ الإنشاء
└── طريقة الدفع
```

### 2.2 سياسة الاحتفاظ بالبيانات (Data Retention)

```
المدة:
├── 📸 الصور: تحذف بعد 30 يوماً من انتهاء الموسم
├── 📋 بيانات الطلب: تحتفظ لموسم واحد فقط
├── 📊 التقارير: مجهولة المصدر (anonymous) للأبد
├── 💳 سجلات الدفع: تحتفظ 7 سنوات (لأغراض ضريبية)
└── 🔐 Audit Logs: تحتفظ سنتين

الحذف:
├── بعد 30 يوم من إعلان النتائج (مايو)
├── العميل يطلب الحذف → نحذف خلال 48 ساعة
└── استثناء: سجلات الدفع باقية (إلزامي قانوني)
```

### 2.3 التشفير والأمان

```
في التخزين:
├── 🗄️ بيانات العميل: مشفرة في PostgreSQL (pgcrypto)
├── 📸 الصور: مشفرة في MinIO (AES-256)
└── 🔐 كلمات السر: bcrypt (rounds 12)

في النقل:
├── 🌐 كل الاتصالات: HTTPS/TLS 1.3
├── 🔄 API ↔ Database: SSL (محلي)
└── 🔁 بين الخدمات: Internal Network فقط

الوصول:
├── 👨‍💼 الموظف: يرى بيانات الطلب فقط (لا يعدّل)
├── 🔧 المطور: لا يرى بيانات حقيقية (فقط test data)
└── 👑 المدير: يرى كل شيء (مسجل في audit log)
```

## 4. بنود الخدمة الأساسية (Terms of Service)

### 4.1 البنود الحرجة (Critical Clauses)

```
البند 1: طبيعة الخدمة
├── Qor3a هي خدمة مساعدة في التقديم،
├── لا تمثل وزارة الخارجية الأمريكية،
└── ولا تضمن القبول أو الفوز.

البند 2: مسؤولية العميل
├── أن يكون مؤهلاً للتسجيل (عمر + تعليم)
├── أن يقدم معلومات صحيحة وكاملة
├── أن يملك جواز سفر ساري المفعول
└── أي خطأ في البيانات يتحمله العميل

البند 3: التفويض
├── العميل يفوض Qor3a بالتقديم نيابة عنه
├── العميل يفوض QOR3A بفتح النتيجة
└── يمكن للعميل إلغاء التفويض قبل التقديم

البند 4: الإلغاء والاسترداد
├── قبل الإدخال الرسمي: استرداد كامل
├── بعد الإدخال الرسمي: لا استرداد
└── إذا رفضت الصورة: استرداد كامل (لم يتم التقديم)

البند 5: الحد الأقصى للمسؤولية
├── قيمة الخدمة فقط (1,000 YR)
├── لا نتحمل مسؤولية رفض التأشيرة
└── لا نتحمل مسؤولية خطأ في بيانات العميل
```
