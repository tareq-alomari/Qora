## 1. Project Overview

**Name**: قرعة (Qor3a)
**Tagline**: اسحب قرعتك - منصة التسجيل في DV Lottery
**Concept**: منصة SaaS تبدأ كـ "نظام آلي للتسجيل في قرعة اللوتري الأمريكي DV Lottery" وتتوسع لبوابة خدمات إلكترونية شاملة (تأشيرات، ترجمة، جوازات).

**Target Audience**: الشباب العربي (18-35 سنة)، المغتربون، الراغبون بالهجرة إلى أمريكا.

**Value**: لا نبيع التسجيل المجاني - نبيع راحة البال، صور احترافية، مراجعة البيانات، فحص النتيجة.

**Record Time**: إتمام التسجيل في < 5 دقائق من الهاتف.

## 3. User Journey - 8 Steps

### The Complete Client Journey

```
🎬 الدخول → 📝 البيانات → 📸 الصورة (AI) → 💳 الدفع
     → 👨‍💼 مراجعة الموظف → 🌐 التسجيل الرسمي
     → 📩 تسليم رقم التأكيد → 🔍 فحص النتيجة
```

### Step Details

#### Step 1: Landing Page (30 sec)
**Goal**: يقنع الزائر بالتسجيل فوراً
**Elemente**:
- Hero section: "سجّل في قرعة اللوتري في أقل من 5 دقائق"
- زر CTA ضخم: "🚀 ابدأ التسجيل الآن"
- 4 خطوات مصورة (بيانات → تصوير → دفع → تأكيد)
- إحصائيات: "أكثر من X مسجل"، "نسبة قبول الصور 95%"
- قسم "لماذا نحن؟" مع 3 مميزات
- تذييل: روابط الشروط + دخول الموظفين

#### Step 2: Auth (45 sec)
**Goal**: إنشاء حساب بسرعة قياسية
**Options**:
- تسجيل برقم الهاتف + OTP (الأفضل)
- تسجيل بالبريد + كلمة مرور
- تسجيل عبر Google/Apple (لاحقاً)

#### Step 3: Personal Info Entry (90 sec) - Main Screen
**Goal**: إدخال البيانات خطوة بخطوة
**Design**: Wizard مع Progress Bar (1/5, 2/5, ...)
**Steps**:
1/5: الاسم الكامل + الجنس + تاريخ الميلاد
2/5: مكان الميلاد (دولة + مدينة)
3/5: المؤهل التعليمي + المهنة
4/5: الحالة الاجتماعية (بيانات الزوج/الأبناء)
5/5: معلومات الاتصال (بريد + هاتف + عنوان)

**Feature**: Auto-Save (كل 10 ثوانٍ) - شريط أصفر "تم الحفظ"

#### Step 4: Photo Capture (60 sec) - The Killer Feature
**Goal**: التقاط صورة مطابقة للمواصفات
**Camera UI**:
```
┌──────────────────────┐
│  × إلغاء             │
│                      │
│  ┌────────────────┐  │
│  │   ┌────────┐   │  │
│  │   │ وجهك   │   │  │
│  │   │ هنا    │   │  │
│  │   └────────┘   │  │
│  │                │  │
│  └────────────────┘  │
│                      │
│  ● خلفية بيضاء      │
│  ● إضاءة جيدة        │
│  ● وجه في المنتصف    │
│                      │
│  [📷 التقاط الصورة]  │
└──────────────────────┘
```
**After Capture**: AI يعالج (3-5 ثوانٍ)
**Result**: إما ✅ مقبولة (تأثير أخضر) أو ❌ مرفوضة مع الأسباب (تأثير أحمر + إعادة)

#### Step 5: Payment (60 sec)
**Goal**: يحوّل المبلغ إلى حسابات قرعة ويرفع الإشعار
**Model**: إيداع يدوي في حساباتنا - **لا API للدفع**، موظف يؤكد لاحقاً
**Elements**:
- عرض ملخص الطلب + السعر (1,000 YR)
- اختيار طريقة الدفع (كريمي، جيب، ون كاش، موبايل موني)
- عرض رقم حساب قرعة في الطريقة المختارة
- يوضّح: "حوّل المبلغ إلى حساب قرعة ثم ارفع صورة الإشعار"
- رفع صورة الإشعار (تصوير أو من المعرض)
- زر "تأكيد الإرسال"

#### Step 6: Confirmation
**Elements**:
- 🎉 "تم استلام طلبك!"
- رقم الطلب
- الحالة: "بانتظار مراجعة الموظف"
- زر تتبع الطلب
- زر العودة للرئيسية

#### Step 7: Client Dashboard
**Elements**:
- قائمة الطلبات (كل طلب + حالته)
- طلب مكتمل: زر "عرض رقم التأكيد"
- طلب معلق: "بانتظار الموظف"
- زر "طلب جديد"
- قسم الإشعارات

#### Step 8: Result Check
**Elements**:
- اختيار طلب مكتمل
- زر "🔍 فحص النتيجة"
- شاشة تحميل (جارٍ الفحص...)
- نتيجة: 🎉 فائز / 😢 لم يفز

## 6. Design Requirements for Stitch

### General
- **Direction**: RTL (Right to Left)
- **Layout**: Mobile-first responsive (320px → 1920px)
- **Style**: نظيف، حديث، Minimal، مساحات بيضاء مريحة
- **Shadows**: Soft shadows (Tailwind shadow-sm / shadow-md)
- **Border Radius**: 12px للبطاقات، 8px للأزرار، 999px للـ badges

### Components Needed
1. **Header** (Logo + روابط + زر تسجيل)
2. **Hero Section** (عنوان + CTA + صورة تمثيلية)
3. **Steps Cards** (4 كروت أفقية)
4. **Progress Bar** (1/5, 2/5, ...)
5. **Form Wizard** (Step-by-step مع Auto-Save)
6. **Camera Frame** (مع إرشادات الوجه)
7. **AI Result Badge** (أخضر مقبول / أحمر مرفوض)
8. **Payment Methods** (3 بطاقات اختيار)
9. **File Upload** (Drag & Drop)
10. **Sidebar Dashboard** (مع عدادات)
11. **Data Table** (قائمة الطلبات)
12. **Order Review Card** (3 أعمدة)
13. **Stat Cards** (4 كروت)
14. **Status Badges** (مراجعة، مكتمل، يحتاج تعديل...)
15. **Modal** (تأكيد إجراء)
16. **Toast Notifications** (نجاح / خطأ)
17. **Timeline** (آخر التحديثات في الطلب)
18. **Footer** (بسيط + روابط)

## 7. UI States to Cover

| State | Example | Emotion |
|-------|---------|---------|
| **Empty** | لا توجد طلبات بعد | تشجيعي + CTA |
| **Loading** | AI يفحص الصورة | متحمس (شريط تقدم متحرك) |
| **Success** | ✅ تم قبول الصورة | احتفال (تأثير أخضر) |
| **Error** | ❌ الصورة مرفوضة | معلوماتي مع سبب واضح |
| **Warning** | ⚠️ الدفع لم يؤكد بعد | تنبيه لطيف |
| **Idle** | نموذج الإدخال | هادئ، مريح للعين |

## 10. Quick Reference - Business Model (السوق اليمني)

| الخدمة | السعر | مقارنة |
|--------|-------|--------|
| 🎯 **تسجيل لوتري (مراجعة + تصوير احترافي + متابعة)** | **1,000 YR** | المكاتب تاخذ 2,000 YR (نحن نصف السعر!) |
| 📸 تصوير احترافي (خدمة منفصلة) | 500 YR | - |
| 🔍 فحص النتيجة الآلي | 500 YR | - |

| 🏆 متابعة الفائزين واستشارات | 3,000-10,000 YR | للفائزين فقط |

### شعارنا في السوق اليمني
> **"نصف السعر، ضعف الخدمة"**
> - المكاتب: 2,000 ريال + دوام محدود + زحمة
> - قرعة: 1,000 ريال + 24/7 + AI + فحص النتيجة

### طرق الدفع في اليمن (إيداع يدوي - لا API)
العميل يحوّل المبلغ إلى حسابات قرعة ثم يرفع صورة الإشعار. الموظف يؤكد يدوياً.

| الطريقة | النوع |
|---------|-------|
| 💳 **كريمي** (Al Kuraimi) | تحويل على حساب قرعة |
| 📱 **جيب** (Jeeb Wallet) | تحويل على محفظة قرعة |
| 💵 **ون كاش** (One Cash) | إيداع في حساب قرعة |
| 📲 **موبايل موني** (Mobile Money) | إيداع عبر MTN/يمن موبايل |
